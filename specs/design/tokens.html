Design tokens

The visual primitives the whole interface is built from — colour, type, spacing and radii — rendered here from their real values. They layer on Bootstrap 5: the brand palette, the gray ramp, the edge-paradigm accent and the confidence palette are defined as CSS custom properties in theme/static/scss/custom.scss, with Inter as the typeface. This page is the human-readable view of those decisions; the journeys bake the same values into their mocks.

Source of truth is the SCSS. Every value below mirrors theme/static/scss/custom.scss (compiled to theme/static/css/bs_main.css — never hand-edit the CSS). When a token changes there, refresh the matching swatch here. This doc renders the tokens; it does not define them.

Edge-paradigm accent

A single purple marks edge objects across the canvas, toggle and forms. It's deliberately orthogonal to both the navy/teal brand and the red→green confidence palette, so it reads as "alternative category," not as brand chrome or a status.

Typography

Inter is the typeface, self-hosted as a variable font (weights 100–900) with a system-sans fallback stack. Headings .h1–.h5 are Bootstrap's scale, recoloured to ink (--dark-blue); below them sit three fluid micro-headings for dense UI text.

Fluid micro-headings (.h6 – .h8)

Each scales with the viewport via clamp(min, ideal, max) and gets lighter as it gets smaller — used for card titles, field labels and dense canvas text. (Resize the window to see them flex.)

Radii

There's no formal radius scale yet — radii are chosen per component (Bootstrap defaults plus a few ad-hoc values). The recurring ones:

4px · inline controls
6px · badges, small btns
8px · cards, panels
15px · large containers
pill · toggles, CTAs

Candidate to tokenise. A small named scale (e.g. --radius-sm/md/lg) would remove the ad-hoc values — a good first move if these tokens ever get a deliberate pass.

⌗ Where the tokens live & how to regenerate for agents

All values are CSS custom properties in theme/static/scss/custom.scss (:root blocks near the top, plus the source-metadata block lower down) layered on Bootstrap 5 imported in theme/static/scss/main.scss. Inter is self-hosted via _inter.scss.

Never hand-edit theme/static/css/bs_main.css — it's compiled. Edit the SCSS and recompile (npm run watch, or one-shot npx sass theme/static/scss/main.scss:theme/static/css/bs_main.css --load-path=node_modules/bootstrap/scss).

This page mirrors those values for human review; it isn't imported anywhere. When a token changes in the SCSS, update the matching swatch / row here so the two don't drift. Each journey's <style> block bakes the same palette in as local CSS vars (see journey-authoring.md).
