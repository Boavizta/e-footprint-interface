Component inventory

The recurring UI primitives the interface is assembled from — the building blocks the journeys compose into flows. Each is shown as a faithful mock (the same markup and CSS the journey docs use) next to a pointer to where it lives in code, so you can see the vocabulary at a glance and jump to the real template.

Faithful, not live: a static page can't render the real Django templates, so these are mocks that mirror them — trust the template.html pointer for the canonical markup. The shared <style> toolkit (from build-a-model) is what keeps the mocks consistent across the hub; the palette is tokens.
