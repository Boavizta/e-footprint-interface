e-footprint-interface — Design hub

The navigable index for design artefacts that engineering reads to know what to build, and design reads to know what already exists. Strategy & rationale: README.

Scope today. The journey set is live — build-a-model (the core editing loop, covering both web and edge) is the reference implementation, its <style> block the shared toolkit, and onboarding, view-results, audit-a-result, compare-models and save-and-load are authored against it. tokens and the component inventory are now stood up too — the design hub is complete.

Journeys

One visual doc per core user flow: who & why · screen sequence · key states · decisions taken. Authored at intent level — faithful to the deployed app. Building a new one? Follow journey-authoring.md (method, template, conventions) — and copy build-a-model.html's style block.

build-a-model Web + edge
The core loop. The three-column canvas → add an object in the side panel → edit it → delete it, with disabled-instead-of-error gating and in-place OOB canvas patches. Covers the Edge toggle and all relationship shapes (list, weighted dict, edge groups). The reference journey.
onboarding First run
First run: home → Start modeling → the template picker (templates / start from scratch / load a file) → land on the canvas → the guided tour and the help drawer. How the canvas itself steers the first move.
view-results
The Results panel: yearly & cumulative emissions charts and the Impact repartition Sankey (with its "Analyse by" chips), the Sources tab, and xlsx export. Trusting the numbers is its companion, audit-a-result.
audit-a-result Methodology
Trace any number to its inputs: provenance (every value's source & confidence, the Sources view) and derivation (the inline formula → the calculus graph → cited inputs). The methodology home — currently distributed across the edit panel and Sources, poised to become first-class.
compare-models
The second model slot and the comparison dashboard: KPI strip, decomposition bars, paired & cumulative emissions charts, assumptions diff. Reached via the ⇄ Compare tab — non-destructive, dismiss in place.
save-and-load
Persistence by file: export / open a single model or the two-model workspace (.e-f.json round-trip including UI config), and the recovery page when a session model fails to deserialize.

Tokens

tokens.html
Colour, type scale, spacing and radii, rendered from the real values in theme/static/scss/custom.scss (--new-primary #2D4675, the gray ramp, --edge-paradigm-accent #7B5DC7, the confidence palette) on top of Bootstrap 5 — with the rationale. The SCSS stays the source of truth; this mirrors it.

Components

component inventory
A catalogue of the recurring UI primitives — object cards, the dynamic form fields, the side panel, modals, the disabled-with-tooltip add button — as faithful mocks (the shared toolkit) each with a code pointer. Grouped Canvas · Chrome · Side panel & forms · Relationships · Overlays.

How journeys fit the workflow

  1. spec.html — what the product does.
  2. journeys/<name>.html — how the user moves through it (this folder).
  3. plan.html — how code delivers it.

Specs and plans are already authored as self-contained HTML in this repo (workflow.md) — journeys are the same posture. See the README for the sync protocol with design work.
