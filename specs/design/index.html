e-footprint-interface — Design hub

The navigable index for design artefacts that engineering reads to know what to build, and design reads to know what already exists. Strategy & rationale: README.

Scope today. The journey set is being authored. build-a-model — the core editing loop, covering both web and edge — is live and is the reference implementation (its <style> block is the shared toolkit). The other four journeys are scoped below with a build order; tokens and the component inventory are pre-staged and stay placeholders until they're stood up.

Journeys

One visual doc per core user flow: who & why · screen sequence · key states · decisions taken. Authored at intent level — faithful to the deployed app. Building a new one? Follow journey-authoring.md (method, template, conventions) — and copy build-a-model.html's style block.

build-a-model Web + edge
The core loop. The three-column canvas → add an object in the side panel → edit it → delete it, with disabled-instead-of-error gating and in-place OOB canvas patches. Covers the Edge toggle and all relationship shapes (list, weighted dict, edge groups). The reference journey.
onboarding 2ndTo author
First run: home → Start modeling → template picker (starter templates / how-to guides / start from scratch) → land on the canvas → the guided tour and the help drawer. (The web/edge toggle belongs to build-a-model, Phase E.)
view-results 3rdTo author
The Results panel: the Sankey with its "Analyse by" chips, the hourly / cumulative / breakdown charts, click-to-explain modals (every value carries its derivation), the Sources tab, and xlsx export.
compare-models 4thTo author
The second model slot and the comparison dashboard: KPI strip, decomposition bars, paired & cumulative emissions charts, assumptions diff table. Reached via the ⇄ Compare tab.
save-and-load 5thTo author
Persistence: download / upload a single model and the two-model workspace (.e-f.json round-trip including UI config), and the recovery page when a session model fails to deserialize.

Tokens

tokens.html Staged
Colors, type scale, spacing, radii. The values live in theme/static/scss/custom.scss (--new-primary #2D4675, the gray ramp, --edge-paradigm-accent #7B5DC7) on top of Bootstrap 5 — this doc will render them and capture the decisions. Placeholder until stood up.

Components

component inventory Staged
A catalogue of the recurring UI primitives — object cards, the dynamic form fields, the side panel, modals, the disabled-with-tooltip add button — rendered from the real templates so it can't lie. Indexed here once it exists.

How journeys fit the workflow

  1. spec.html — what the product does.
  2. journeys/<name>.html — how the user moves through it (this folder).
  3. plan.html — how code delivers it.

Specs and plans are already authored as self-contained HTML in this repo (workflow.md) — journeys are the same posture. See the README for the sync protocol with design work.
