Variation 1 — Compare inside the Results panel

Design variation · companion to spec.html · 2026-06-11

Not chosen. Variation 2 (comparison dashboard) is the v1 layout — see the canonical, simplified design in spec.html §4.2. This file is kept as a record of the design exploration.

Concept

Comparison is a results activity, so it lives where results already live: the Results panel that slides over the builder. No new top-level surface. The panel gains a scope selector: each model individually, or "Both (compare)". The existing Results / Sources duality is preserved and becomes the comparison's structure:

  • Results + Both → output comparison (totals, charts, Sankeys).
  • Sources + Both → input comparison: the assumptions-diff table ("what differs between the two models"). The Sources tab already answers "what numbers went in" for one model; in compare scope it answers "which inputs differ".

Assessment

StrengthsZero new navigation surface; comparison inherits every panel behaviour (open/close, mobile layout, future panel features) for free; the Sources→diff mapping is conceptually elegant; switching scope keeps the user's place (same tab, same scroll intent).
WeaknessesComparison content is squeezed into a panel designed for one model's charts — side-by-side Sankeys and the diff table want width; "Both" as a third pill is less discoverable than a dedicated tab; no natural place for a delta-decomposition story (panel invites glancing, not reading).
Best whenComparison is treated as a quick check while editing ("did my change help?") rather than a deliverable to study and share.
