Model comparison — Backlog

Deferred ideas · companion to spec.html · 2026-06-15

Ideas explored during the spec/design pass that are deliberately out of the first iteration. Each is worth doing later; none blocks v1. The design work already done for several of these lives in the variation files, linked below, so we don't re-derive it.

1. Sankey repartition comparison

Each Sankey analysis card becomes a pair of diagrams driven by one shared settings form (phase filter, aggregation threshold, analyse-by chips), drawn at a shared flow scale (identical px-per-kg) so the smaller model is visibly thinner; hovering a node/flow highlights its counterpart and shows both values + delta.

Why deferred: the richest single block, and the per-category delta decomposition in v1 already answers "where does the difference come from" without it. Design fully mocked in variation 2 ("Impact repartition" block, hover the flows).

Rejected sub-idea: a single merged "diff Sankey" — fragile when the two models' structures diverge.

2. Per-year / differing-timespan normalization

v1 computes every difference as a full-timespan sum and plots time-series on a shared calendar axis. That is honest only when the two models cover the same period. For models of different durations, two refinements were designed:

  • Aligned-starts axis — x-axis = "years since each model's own start", so a 2-year and a 3-year model line up at year 1.
  • Per-year-average aggregates — every KPI delta and decomposition bar computed on a per-year basis and explicitly labeled, with each model's card still showing its own absolute total and span.

Why deferred: user chose full-timespan sums for v1 simplicity. Design lives in the rev-2 spec history (§4.5 had a T1/T2/T3 toggle mockup).

3. Δ-only per-period chart mode

A toggle on the emissions-over-time chart that collapses the paired bars into one diverging bar per time bucket (variant minus reference per period).

Why deferred: it is a per-period difference, in tension with v1's full-timespan-sum framing. Mocked in variation 2 ("Δ only" radio).

4. Cause linking (input diffs → impact categories)

Inside each decomposition step, surface the related input differences — e.g. linking "Network usage −240 kg" to "data transferred per call: 800 MB → 250 MB".

Honesty constraint carried forward: this can only be presented as "related differences", matched through the object an input belongs to. It must never claim computed attribution ("X kg of this step come from this input") — that needs sensitivity analysis, which is out of scope.

Why deferred: user moved it to backlog. Design lives in variation 3 (waterfall step accordions).

5. Verdict sentence & delta-first narrative ordering

An opening plain-language verdict ("variant emits 1.11 t less over 2026–2028") above a clickable waterfall of why, with full charts collapsed below — answer-first reading for the non-technical persona.

Why deferred: needs careful wording for the mixed-sign case (lower usage but higher fabrication) before it reads honestly. Design lives in variation 3.

6. Comparison-specific xlsx export

A single spreadsheet of the comparison (both totals, the category decomposition, the assumptions diff). v1 keeps each model's own xlsx export only.

7. More than two models

Three or more models compared at once. v1 is strictly two. This touches the switcher, the KPI strip layout, and the diff table shape, so it is a genuine follow-up rather than a tweak.
