Variation 3 — Delta-first story

Design variation · companion to spec.html · 2026-06-11

Not chosen for v1. Variation 2 (comparison dashboard) is the v1 layout. The delta-first verdict + waterfall and the cause-linking shown here are deferred to backlog.html §4–5. Kept as a record of the design exploration.

Concept

Built for the primary persona (non-technical product person): the comparison page reads answer first, evidence below. It opens with a one-sentence verdict, then a waterfall showing what explains the difference, where each step is clickable and unfolds the evidence behind it — including the input differences that caused it. The full chart gallery (as in Variation 2) is still there, but collapsed at the bottom rather than leading. Access is the same third top-level tab as Variation 2 — this variation differs in content order and narrative, not in navigation.

Honest caveats

  • Category attribution is exact: the waterfall steps are the per-category emission deltas, which are additive — no estimation involved.
  • Cause linking is heuristic: an input difference is linked to a waterfall step through the object it belongs to, but one input can affect several categories (data transferred per call moves both Network and Servers). The UI must present these as "related differences", never as computed attributions ("X kg of this step come from this input" is not claimed — that would require sensitivity analysis, out of scope).

Assessment

StrengthsAnswer-first reading order matches how a product team actually consumes a comparison; the waterfall connects outputs to causes — the unique value a modeling tool can add over two screenshots; degrades gracefully (charts still available below).
WeaknessesMost novel = most design risk; needs careful wording to keep cause-links honest (see caveats); more build effort than reusing existing chart blocks; the verdict sentence needs a neutral form when neither model is obviously "better".
Best whenThe comparison's job is to support a decision and explain it — the typical "baseline vs variant" scenario.
