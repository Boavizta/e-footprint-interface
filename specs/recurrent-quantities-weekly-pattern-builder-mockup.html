Weekly Pattern Builder

New builder for ExplainableRecurrentQuantities fields — lets users describe a 168-hour weekly quantity as day profiles with baselines + ranges, instead of a single constant.

Try the interactions

  • Toggle Constant value ↔ Weekly pattern at the top — each builder's inputs are preserved while switching.
  • Click a day on one profile — it steals the day from whichever profile previously owned it.
  • Edit a range's start hour so it overlaps another range, then click elsewhere — inline error appears, row stays where it is.
  • Edit a range so it moves chronologically (e.g. 07–12 → 19–22), then blur — the row auto-reorders into its slot.
  • Click Add range — pre-filled with the first free slot. When the 24 hours are fully covered, the button disables.
  • Click Add profile — appends a new empty profile. Steal days from existing profiles to populate it.
  • Remove a profile that owns days — those days become unassigned and Save is blocked until reassigned.
  • Type a negative number into a baseline or range value — refused because can_be_negative = False on this attribute.

Not in this mockup

  • Live 168-hour chart preview (deferred to a later iteration — current read-only chart keeps working after save).
  • Drag-and-drop timeline / painted 7×24 grid (out of scope for V1).
  • CSV import, sub-hour granularity, cross-profile templates.

Attribute in this demo: ram_needed_per_job, unit concurrent, can_be_negative = False.
