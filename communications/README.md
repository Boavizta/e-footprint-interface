# Communications — interface side

Working drafts for communications whose **subject is the e-footprint interface itself**: UI feature announcements, walkthroughs, release notes.

## Where to find foundations

Strategy, messaging one-pager, pitch deck outline, and the **single editorial calendar** live in the library repo:

- `e-footprint/communications/strategy.md`
- `e-footprint/communications/messaging-one-pager.md`
- `e-footprint/communications/editorial-calendar.csv`

Add new interface-side drafts to `articles/`, `linkedin/posts/`, or `release-notes/` here, then register them in the calendar over there.

## What lives here

| Folder | Contents |
|---|---|
| `articles/` | Articles whose dominant subject is an interface feature (e.g. onboarding templates, edge toggle UX, guided tour) |
| `linkedin/posts/` | LinkedIn posts whose dominant subject is an interface feature |
| `release-notes/` | Release notes tied to interface versions |

PS-branded client case studies do **not** live here — they are PS-owned and stay in PS systems.

Default rule for ambiguous pieces: wherever the dominant subject lives. Tool/method content goes to the library repo's `communications/` folder.
